# DESK collector dependencies — EXACT pins (Phase 15A / hardening H1).
#
# WHY PINNED: these installs run in CI every 5-15 min with
# SUPABASE_SERVICE_ROLE_KEY (RLS-bypass) and DESK_DB_URL in the environment.
# Floating ranges (>=) would pull the latest release on every run, so a single
# malicious/compromised PyPI upload could execute at install time and exfiltrate
# those credentials. Exact "==" pins freeze the install to the reviewed set;
# an upgrade becomes a deliberate, reviewable commit instead of a silent event.
#
# Every version below is verbatim from research/freeze_snapshot.txt (a real
# `pip freeze` of the environment the collectors run in). Do not invent versions.
#
# UPDATE PROCEDURE (one dep at a time, deliberately):
#   1. Bump the single line you intend to change.
#   2. Recreate a clean venv and `pip install -r requirements.txt`.
#   3. `pip freeze > research/freeze_snapshot.txt` and re-pin any transitive
#      versions that moved (from the new snapshot, verbatim).
#   4. Run the collectors / tests, then commit the requirements.txt +
#      freeze_snapshot.txt change together with a note on what was bumped and why.
#
# HASH PINNING: `pip freeze` carries no hashes and we pin offline, so this is
# exact-version only for now. Hash pins (`--require-hashes`, closing the
# version-reuse gap) are a follow-up — generate with `pip-compile --generate-hashes`
# or `pip hash`; do not block H1 on it.
